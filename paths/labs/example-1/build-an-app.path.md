# Example Learning Path: How to Build an App

This path will use a tiny task app to teach how a user action travels through
an application: interface, frontend, API, backend, database, and response.

The design references for the first prototype are collected in the
[inspiration library](./inspiration.guide.md).

---

## Resources and inspiration

These examples are optional reference points for the path. Open one when you
want to see a particular interaction or explanation pattern in action.

### Try a focused decision

[Eggs](https://eggs.pplx.app/) compares two cooking methods side by side and
turns a choice into an immediate action. Borrow this pattern for deciding
whether a task belongs in the frontend or backend.

### Assemble a system

[Don't Starve Together Squad Planner](https://dontstarvetogether.pplx.app/)
combines roles, constraints, coverage gaps, and recommendations. Borrow this
pattern for assembling the parts of Pocket Tasks and finding what a feature is
missing.

### Compare capabilities

[Markdown vs. HTML](https://md-html.pplx.app/) explains a technical difference
with a capability matrix and a live side-by-side example. Borrow this pattern
for comparing browser-only behavior with work that requires an API or server.

### Explore a visual model

[Shape of the Day](https://shapeoftheday.pplx.app/) shows overlapping contexts
across a timeline. Borrow this pattern for showing how one click touches user
experience, data, security, and performance.

[The Dialectical Spiral](https://hegelianspiral.pplx.app/) uses a repeating,
pauseable visual cycle to explain an abstract idea. Borrow this pattern for
the request → processing → response loop.

### Follow a longer explanation

[The AI Bubble](https://theaibubble.pplx.app/) breaks a complex argument into
chapters, evidence cards, diagrams, and small calculations. Borrow this
pattern for introducing the application layers one chapter at a time.

[The Daily Wire AM](https://thedailywire-am.pplx.app/) organizes many sources
into topic sections, summaries, and outbound references. Borrow this pattern
for an optional technical-reference shelf that stays separate from the core
lesson.

For the first prototype, start with [Eggs](https://eggs.pplx.app/), [Don't
Starve Together Squad Planner](https://dontstarvetogether.pplx.app/), and
[Markdown vs. HTML](https://md-html.pplx.app/). The full notes and “what to
borrow” guidance live in the [inspiration library](./inspiration.guide.md).
