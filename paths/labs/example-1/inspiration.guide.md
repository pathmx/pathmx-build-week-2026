# Inspiration Library: How to Build an App

This guide collects interaction and storytelling references for the Pocket
Tasks learning path. These are design inspirations, not required learner
reading. Borrow the useful move from each example and adapt it to the lesson.

---

## Interaction patterns

### [Eggs](https://eggs.pplx.app/)

**Design move:** Present a small decision, compare alternatives side by side,
and give the learner an immediate action such as starting a timer.

**Pocket Tasks use:** Compare whether work belongs in the frontend or backend,
then let the learner predict the result of each choice.

**Do not copy:** Turn every lesson into a configuration screen.

### [Don't Starve Together Squad Planner](https://dontstarvetogether.pplx.app/)

**Design move:** Let people assemble a team from roles, reveal coverage gaps,
and make recommendations based on the current state.

**Pocket Tasks use:** Have the learner assemble interface, frontend, API,
backend, and database components, then identify which responsibility is
missing from a feature.

**Do not copy:** Add planning complexity that does not support the learning
goal.

### [Shape of the Day](https://shapeoftheday.pplx.app/)

**Design move:** Show several overlapping contexts on one visual timeline and
let the reader change the view.

**Pocket Tasks use:** Visualize how a single click moves through user
experience, data, security, and performance concerns over time.

**Do not copy:** Use visual density when a simple sequence would be clearer.

### [The Dialectical Spiral](https://hegelianspiral.pplx.app/)

**Design move:** Explain an abstract idea with a repeated visual cycle,
progressive labels, and pause or restart controls.

**Pocket Tasks use:** Represent request → processing → response as a cycle the
learner can inspect one stage at a time.

**Do not copy:** Force a metaphor to carry more technical detail than it can
support.

## Explanation and narrative patterns

### [Markdown vs. HTML](https://md-html.pplx.app/)

**Design move:** Compare two systems through a capability matrix and a live
example of the same intent.

**Pocket Tasks use:** Compare what the browser can do alone with what requires
an API, server, or database.

**Do not copy:** Present a matrix without giving the learner a decision to
make.

### [The AI Bubble](https://theaibubble.pplx.app/)

**Design move:** Break a complex argument into short chapters, persistent
visual models, evidence cards, and small calculations.

**Pocket Tasks use:** Give each application layer its own chapter while
keeping the complete request journey visible as the learner progresses.

**Do not copy:** Let supporting detail overwhelm the central learning move.

### [The Daily Wire AM](https://thedailywire-am.pplx.app/)

**Design move:** Organize a large set of material into topic sections,
scannable summaries, and clearly attributed outbound sources.

**Pocket Tasks use:** Keep deeper technical references in an optional shelf so
learners can follow curiosity without leaving the main path.

**Do not copy:** Make the learner browse a digest before they can continue the
lesson.

## Recommended first-prototype references

Use these three as the primary design references for the first Pocket Tasks
prototype:

1. **Eggs** for focused decisions and immediate feedback.
2. **Don't Starve Together Squad Planner** for assembling components and
   revealing gaps.
3. **Markdown vs. HTML** for side-by-side explanation and capability mapping.

Use **The Dialectical Spiral** for pacing, **Shape of the Day** for visual
systems thinking, and **The AI Bubble** or **The Daily Wire AM** for optional
chapter and reference patterns.

These links are inspiration only. The learner-facing path should remain
understandable as plain Markdown and should expose a reference selectively
when it supports the current activity.
